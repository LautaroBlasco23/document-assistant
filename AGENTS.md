# AGENTS.md — Document Assistant

## Repository layout

```
backend/          # Python FastAPI backend (pyproject.toml + uv.lock here)
frontend/         # React + TypeScript + Vite SPA
config/           # YAML config (default.yml) — NOT inside backend/
scripts/          # Dev automation scripts
data/raw/         # Place PDFs/EPUBs here before ingesting
data/output/      # Manifests per ingested document
docker-compose.yml
Makefile
```

## Configuration

- **Config file**: `config/default.yml` at project root (not inside `backend/`).
- **Env overrides**: prefix `DOCASSIST_`, nested delimiter `__` (e.g. `DOCASSIST_GROQ__API_KEY`).
- **`.env`**: auto-generated by `bash scripts/setupEnv.sh`. Never commit it.
- **LLM provider**: set via `DOCASSIST_LLM_PROVIDER` env var or `llm_provider` in config. Supported: `groq`, `ollama`, `openrouter`, `huggingface`.

## Developer commands

```bash
# Full dev stack (recommended)
make start

# Frontend only, mock data (no backend needed)
VITE_MOCK=true npm run dev    # or: make mock

# Backend + frontend on host, PostgreSQL in Docker
docker compose up -d postgres
cd backend && uv sync
cd backend && uv run uvicorn api.main:app --port 8000
cd frontend && npm install && npm run dev   # port 5173, proxies /api to 8000

# CLI (run from backend/)
uv run python -m cli.main check              # health check
uv run python -m cli.main ingest ../data/raw/file.pdf
uv run python -m cli.main summarize "book" 1  # chapter 1, 1-based
uv run python -m cli.main generate-md "book" 1 # summary + flashcards
uv run python -m cli.main prune               # remove orphaned manifests

# Tests
cd backend && uv run pytest                  # unit only
cd backend && uv run pytest -m integration   # requires Docker running
cd backend && uv run ruff check .            # lint
cd backend && uv run ruff check --fix .

# Frontend
cd frontend && npm run dev       # dev server port 5173
cd frontend && npm run build      # build (tsc + vite)
cd frontend && npm run type-check # TypeScript only
```

## Key conventions

- **Chapter numbers**: 1-based in CLI args and API; routers convert to 0-based before calling the application layer.
- **`core/` layer rule**: `core/model/` and `core/ports/` must not import from `infrastructure/` or `application/`.
- **Integration tests**: marked `@pytest.mark.integration`, skipped when PostgreSQL is unreachable. Run `docker compose up -d postgres` first.
- **Schema migrations**: idempotent SQL migrations in `infrastructure/db/migrations/`. Applied automatically on startup.
- **Prompts**: all agent system prompts in `backend/application/prompts.py`.
- **Generated content**: summaries and flashcards are persisted in PostgreSQL (not just files). Manifests go to `data/output/<book>/manifest.json`.

## Docker-specific quirks

- **Ollama on host**: in `docker-compose.yml`, `DOCASSIST_OLLAMA__BASE_URL=http://host.docker.internal:11434` — Ollama must be running on the host machine, not in a container.
- **Config mount**: `backend/Dockerfile` copies `config/` to `/config/` inside the image. `infrastructure/config.py` resolves the project root as three levels up from that file, so it finds `/config/default.yml` in the container.
- **Docker env vars**: `make start` calls `scripts/setupEnv.sh` which syncs `POSTGRES_PASSWORD` into `DOCASSIST_POSTGRES__PASSWORD` so the host-backend uses the same credential as the container.

## Frontend dev server

- Port 5173. Vite proxies `/api` to `localhost:8000`.
- Only works when `VITE_MOCK` is unset or `false` and the backend is running.
- Production build outputs to `frontend/dist/`, served by nginx in Docker.

## Dependency management

- **Backend**: `uv sync` from `backend/`. Python 3.12+.
- **Frontend**: `npm install` from `frontend/`. Node 18+.
- **Docker images**: pinned versions in `docker-compose.yml` (e.g. `postgres:17.4-alpine`).
